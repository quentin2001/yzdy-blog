## **1. 项目背景**

传统 ToB 销售面临三大问题：

1. **产品繁杂**，几百页参数 / 版本难以记忆
2. **响应慢**，需要查询 CRM、库存、价格、审批
3. **竞品多**，需要随时给出对比和解决方案

我的目标是：

> **基于 CherryStudio + MCP + PostgreSQL 构建一个真正能帮销售工作的 AI Agent，让销售像带着一个“售前小伙伴”一样工作。**

---

## **2. 系统架构（面试亮点）**

```
User
 ↓
CherryStudio Agent
 ↓（MCP）
Node.js MCP Server（我自研）
 ↓（SQL）
PostgreSQL（产品 / 库存 / 客户画像 / 竞品）
```

三大亮点：

---

### **亮点 1：使用 MCP 作为工具调用标准化协议**

我不是把数据库连接硬写到聊天机器人里，而是：

* 使用 Model Context Protocol（MCP）
* 让模型以 **工具的形式** 调用 PostgreSQL
* 每个工具有输入 Schema、输出 Schema、更安全、更标准

这种架构非常适合企业级扩展。

---

### **亮点 2：10 个强业务工具，把销售流程拆成可组合模块**

包括：

* 产品信息查询
* 库存查询
* 折扣范围判断
* 总价计算
* 竞品对比
* 客户画像
* 销售机会
* 交付时间估算
* 销售建议生成

所有工具均以 SQL + 业务逻辑组合实现。

---

### **亮点 3：通过 CherryStudio 的 Agent 调度，实现自动决策**

模型不仅查询数据，也会：

* 自动决定调用哪些工具
* 将工具结果整合
* 生成完整解决方案

例如一句话：

> “客户在华东买 80 套，给我报价策略。”

模型会自动：

1. 调库存
2. 调交付日期
3. 调折扣范围
4. 调价格表
5. 调审批规则
6. 输出策略和成交价
