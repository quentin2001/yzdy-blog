我想用cherrystudio来实现一个B2B 售前解决方案专家助手 (Sales Copilot)，利用cherrystudio自带的知识库和我已经配置好的postgre MCP服务。

在 ToB 企业（如 SaaS、高端制造、医疗器械）中，传统销售人员经常在外面跑客户，他们面临一些痛点，这里我举两个例子：
1. 产品太复杂：记不住几百页的技术参数和竞品对比（需要 RAG 知识库）。
2. 响应要及时：客户问“我要买 50 套 A 型号，有没有货？最低折扣多少？”，销售需要查 CRM、查库存、算价格（需要 Agent 调用工具）。
场景小例子：销售在 cherrystudio chatbot里问：“客户是制造业的，想买我们要发布的 V3.0 系统，能不能把我们和竞品 X 的对比发我？另外查一下华东仓还有多少库存？” chatbot 不仅给出了对比文档摘要，还直接调用 MCP 告诉他库存剩 120 件。
当然还有更多的痛点，因为我不是销售人员，所以需要你从专业高级销售角度帮我挖掘更多痛点出来

然后我希望你能帮助我一步一步搭建出这个案例模板，我对于这个案例有以下几个要求，你需要严格遵守：
1. 该chatbot的使用者是销售团队的成员，不是客户
2. 这个案例不使用多Agent，不需要有多Agent方案和顾虑
3. 模拟数据可以多一些，字段类型也可以多一些
4. 对于知识库的构建所用到的材料，你可以帮我找一些材料，或者模拟生成一些材料
5. 需要有一个地方突出一下对MCP的使用，因为这个案例我想拿去面试用，面试过程中有可能会用到
